1. ООП - методология для построения программы из объектов, которые могут обмениватсья методами, свойствами друг с другом. Работает на 4 концепциях - полиморфизм (одни и те же методы работают по разному), наследование (создание одного класса на основе другого), инкапсуляция (запирает свойства внутри класса, реалует доступ получения к ним через геттер и сеттер), абстракция
2. Замыкание - эффект, возникающий, когда функция уже выполнилась, а ссылка на ее окружение до сих пор есть, то есть метод использует переменную внутри функции, что уже выполнилась
3. prototype - способ наследования в объектах. Когда объект не находит свойство у себя - ищет у родителяобъекты, что есть в каждой функции. [[Prototype]] - ссылка на родителя, по факту являтся объектом. Контруктор - доступ к свойствам фукнции контруктора.

еще короче
ООП — программа из объектов.
Замыкание — функция помнит переменные снаружи.
Prototype — объекты могут наследовать друг у друга.
